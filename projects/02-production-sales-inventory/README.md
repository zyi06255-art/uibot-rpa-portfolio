# 产销存日报自动化 / Production-Sales-Inventory Daily Report

## 项目简介

自动从业务系统中抓取生产、销售、库存数据，经过运算处理后生成当日产销存日报 HTML 报表，并通过消息系统分发给相关部门。

## 技术架构

```
数据下载 → 数据运算与处理 → 流数据存储 → HTML报表生成 → 消息分发
    ↓              ↓              ↓            ↓             ↓
 下载文件.task   数据运算.task   流数据.task   Img_HTML.py   消息分发.task
```

## 核心流程

| 流程块 | 功能 | 要点 |
|--------|------|------|
| 下载文件 | 从数据源下载当日产销存 Excel | 文件存在性检查 |
| 创建状态 | 在追踪表中初始化流程状态 | 幂等防重 |
| 初始化 | 环境检查与资源准备 | 异常兜底 |
| 数据运算与处理 | 核心计算逻辑：产量/销量/库存汇总 | 多 Sheet 动态处理 |
| 反写状态 | 将处理结果写回源系统 | 事务性保证 |
| 流数据存储与更新 | 将结果持久化到数据库 | 增量更新 |
| 消息与文件分发 | 生成 HTML 报表并推送通知 | Img_HTML.py |
| 异常处理 | 捕获全流程异常并记录日志 | 优雅失败 |

## 技术亮点

### HTML 报表生成

`Img_HTML.py` 和 `imghtml.py` 负责将数据渲染为格式化的 HTML 日报，支持：
- 动态表格生成
- 条件格式化（异常数据高亮）
- 响应式布局（适配桌面/移动端查看）

### 多步流程编排

8 个流程块形成一条完整管线，每个节点独立执行且可单独重试。通过状态表实现节点间的依赖管理。

## 文件说明

| 文件 | 角色 |
|------|------|
| `下载文件.task` | 从数据源下载产销存 Excel 文件 |
| `数据运算与处理.task` | 核心业务计算逻辑 |
| `流数据存储与更新.task` | 数据持久化 |
| `反写状态.task` | 结果回写 |
| `消息与文件分发.task` | 报表生成与推送 |
| `异常处理.task` | 全局异常捕获 |
| `Img_HTML.py` | HTML 报表渲染 |
| `imghtml.py` | HTML 模板工具 |
| `产销存日报.html` | 报表输出模板 |
