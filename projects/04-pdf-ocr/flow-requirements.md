## 流程步骤

详细描述每个步骤的具体操作和验证方式：

```
示例：
第一步：打开本地Chrome浏览器，访问目标网站 http://192.168.10.188:3001/

第二步：输入"账号"和"密码"，再点击"立即登录"按钮，进行表单登录，等待主页是否已正常加载，可以用"工作台"菜单是否存在来判断是否成功加载，否则抛出登录异常。注意：账号和密码先以流程图变量代替，到时我自己填写变量值

第三步：在主页点击导航菜单"我的单据"，看到切换到"费控管理"Tab页签，则代表导航正确，否则抛出导航异常

第四步：点击"新建"按钮，弹出模态窗口，在"财务报销"分组中点击"差旅费报销"选项，此时模态窗口关闭

第五步：填写单据的概述部分，在"标题"输入框中填写"差旅费报销"，在"预算占用部门"下拉列表中选择"大客户业务中心"，"报销公司主体"下拉列表中选择"来也科技(北京)有限公司"，"描述"输入框中输入"test",在"单据内发票类型"下拉列表中选择"电子发票"

第六步：填写单据的明细部分，在" 费用明细"表格区中点击"添加"按钮，此时右侧弹出详情窗口，在"消费城市"输入框中填写"北京"，"金额"字段中输入数值388，在"发票"字段中点击上传按钮，在打开文件会话框中，选择路径"xxxxx",点击"保存"按钮，完成单据填写

第七步：点击"提交送审"按钮；再点击登录用户头像，在弹出的菜单中点击"退出登录"选项，确认退出登录。
```

## 界面元素命名规则

- 步骤中出现的按钮、输入框、下拉选项等目标元素，使用自定义界面元素名称代替，由开发者单独手动拾取元素属性
- 名称格式：`ui_` + 功能描述
- 示例：`ui_login_button`、`ui_title_input`

## 变量定义（可选）

如需定义特殊变量，请参考以下格式：

### 流程图变量
| 名称 | 描述 | 默认值 |
|---------|---------|--------|
| g_h_web | 保存浏览器对象句柄，供后续流程块使用 | null |

### 流程输入变量
| 变量名称 | 变量类型 | 描述 | 默认值 |
|---------|---------|---------|--------|
| login_account | string | 登录账号 | user |
| login_password | credential | 登录密码（凭据类型） | - |
| invoice_file_path | string | 发票文件路径 | C:\Users\xxx\invoice.pdf |

### 流程输出变量
| 变量名称 | 默认值 | 描述 |
|---------|---------|---------|
| task_result | null | 任务执行结果，成功为true，失败为false |
